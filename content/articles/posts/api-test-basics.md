# 接口测试基础入门

## 1. 接口测试在测什么

接口测试关注的是服务之间的数据交换是否符合约定，重点通常包括：

- 请求参数是否完整
- 返回状态码是否正确
- 响应结构是否稳定
- 核心字段的业务含义是否符合预期

## 2. 一条接口用例最少要覆盖什么

建议先覆盖这几类：

1. 正常输入
2. 必填参数缺失
3. 参数类型错误
4. 权限或鉴权失败
5. 边界值

## 3. 常见断言思路

- 状态码断言
- 响应字段断言
- 数据库或下游结果断言
- 幂等性断言

## 4. 面试中经常会问

“如果接口返回成功，但业务数据错了，这算接口通过吗？”

标准回答通常是：不算。因为接口测试不仅验证协议层成功，还要验证业务结果正确。
